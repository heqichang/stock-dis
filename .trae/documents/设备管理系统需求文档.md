# 公司设备管理系统需求文档

## 1. 项目概述

### 1.1 项目背景
为满足行政人员对公司设备进行高效管理的需求，开发一套单机版设备管理系统。系统支持通过扫码枪扫描设备唯一识别码快速定位设备，并对设备状态进行管理。

### 1.2 项目目标
- 实现设备信息的录入、查询、修改、删除
- 支持条形码（Code 128）打印与扫描识别
- 设备状态管理（使用中 / 报废）
- 单机运行，无需网络依赖

---

## 2. 技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| 开发语言 | C++17 | 原生开发，性能优异，无运行时依赖 |
| 桌面框架 | Win32 API | Windows 原生 GUI，兼容 Win7 及以上 |
| UI 控件 | 自绘控件 + 通用控件 | ListView、Edit、Button 等 Win32 通用控件，配合自绘实现现代外观 |
| 构建工具 | CMake + MSVC | CMake 管理构建，MSVC 编译器（v141_xp 工具集兼容 Win7） |
| 数据库 | SQLite 3 | 轻量级本地数据库，C/C++ 原生接口，零配置 |
| 条码生成 | Zint | 开源 C 语言条码库，支持 Code 128 等多种格式 |
| 条码打印 | Win32 GDI 打印 API | 直接调用系统打印驱动，精确控制打印布局 |
| 数据导出 | nlohmann/json + libxlsxwriter | JSON 导入导出 + Excel 文件生成 |

## 3. 功能需求

### 3.1 设备管理

#### 3.1.1 设备信息结构

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | INTEGER | 是 | 主键，自增 |
| name | TEXT | 是 | 设备名称 |
| code | TEXT | 是 | 唯一识别码（条形码内容） |
| status | TEXT | 是 | 状态：使用中 / 报废 |
| params | TEXT | 否 | 设备参数信息（JSON 字符串） |
| created_at | DATETIME | 是 | 创建时间 |
| updated_at | DATETIME | 是 | 更新时间 |

#### 3.1.2 功能列表

1. **设备列表**
   - 分页展示所有设备
   - 支持按名称、识别码搜索
   - 支持按状态筛选
   - 列表显示：名称、识别码、状态、更新时间

2. **新增设备**
   - 填写设备名称
   - 系统自动生成唯一识别码（或手动输入）
   - 填写设备参数（JSON 格式，提供可视化表单）
   - 保存后自动跳转打印页面

3. **编辑设备**
   - 修改设备名称
   - 修改设备参数
   - 不支持修改识别码（识别码一旦生成不可变更）

4. **删除设备**
   - 支持单条删除
   - 删除前需二次确认

5. **设备详情**
   - 展示完整设备信息
   - 展示条形码图片
   - 支持重新打印条形码

### 3.2 状态管理

- **使用中**：设备正常在役
- **报废**：设备已报废，不可再转为使用中
- 状态变更操作：
  - 使用中 → 报废（需确认，报废不可逆）

### 3.3 条形码功能

#### 3.3.1 条形码生成
- 格式：Code 128
- 内容：系统生成的唯一识别码（建议 8-12 位字母数字组合）
- 生成时机：新增设备时自动生成

#### 3.3.2 条形码打印
- 打印尺寸：50mm × 30mm 标签纸
- 打印内容：条形码 + 设备名称 + 识别码文字
- 支持预览后打印
- 支持批量打印（选中多个设备）

#### 3.3.3 条形码扫描识别
- 扫描枪输入模拟键盘输入
- 全局监听扫描输入，自动识别并查询设备
- 扫描后跳转设备详情页或操作弹窗

## 4. 界面设计

### 4.1 页面结构

```
├─ 首页（设备列表）
│   ├─ 搜索栏
│   ├─ 状态筛选
│   ├─ 新增设备按钮
│   └─ 设备表格（含操作列）
├─ 新增/编辑设备页
│   ├─ 设备名称输入
│   ├─ 识别码展示（编辑时不可改）
│   ├─ 设备参数表单（动态 JSON 编辑器）
│   └─ 保存/取消按钮
├─ 设备详情页
│   ├─ 设备信息卡片
│   ├─ 条形码展示
│   ├─ 打印按钮
│   └─ 状态操作按钮
└─ 打印预览页
    ├─ 条形码预览
    └─ 打印按钮
```

### 4.2 关键交互

- **扫码枪扫描**：任何页面下，扫描枪输入识别码后，系统自动查询并弹出设备操作卡片
- **状态变更**：设备详情页点击"报废"按钮，二次确认后变更状态

## 5. 数据存储

### 5.1 数据库表结构

```sql
CREATE TABLE devices (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name TEXT NOT NULL,
    code TEXT NOT NULL UNIQUE,
    status TEXT NOT NULL CHECK(status IN ('使用中', '报废')),
    params TEXT DEFAULT '{}',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_devices_code ON devices(code);
CREATE INDEX idx_devices_status ON devices(status);
```

### 5.2 数据备份
- 支持导出所有设备数据为 JSON/Excel 文件
- 支持从 JSON/Excel 文件导入设备数据

## 6. 非功能需求

### 6.1 性能
- 页面加载 < 1s
- 设备列表支持 10000+ 条数据流畅滚动

### 6.2 兼容性
- 支持 Windows 7/10/11
- 支持主流 USB 扫码枪（模拟键盘输入模式）
- 支持主流标签打印机

### 6.3 安全性
- 单机运行，无外网依赖
- 数据存储在本地 SQLite 文件
- 无额外运行时依赖，单 EXE 绿色部署
